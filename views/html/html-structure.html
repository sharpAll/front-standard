HTML规范 - 整体结构

HTML基础设施
  • 文件应以“<!DOCTYPE html>”首行顶格开始。
  • 必须申明文档的编码charset，且与文件本身编码保持一致，例如<meta charset="utf-8"/>。
  • 根据需求填写keywords和description。
  • 页面title概述此页面内容。
                            
<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8"/>
<title></title>
<meta name="keywords" content=""/>
<meta name="description" content=""/>
<meta name="viewport" content="width=device-width"/>
<link rel="shortcut icon" href="img/favicon.ico"/>
<link rel="apple-touch-icon" href="img/touchicon.png"/>
<link rel="stylesheet" href="css/style.css"/>
</head>
<body>
</body>
</html>
结构顺序和视觉顺序基本保持一致
  • 按照从上至下、从左到右的视觉顺序书写HTML结构。
  • 用div代替table布局，可以增加渲染速度，使HTML更具灵活性，也方便利用CSS控制。
  • table用于表现具有明显表格形式的数据，不建议用于布局。
结构、表现、行为三者分离
  • 使用link将css文件引入，并置于head中。
  • 使用script将js文件引入，并置于body底部。
保持良好的简洁的树形结构
  • 每一个块级元素都另起一行，使用Tab缩进对齐。
  • 在大的模块之间用空行隔开，并添加注释使模块更清晰。
                            
<body>
<!-- 侧栏内容区 -->
<div class="m-side">
    <div class="side">
        <div class="sidein">
            <!-- 热门标签 -->
            <div class="sideblk">
                <div class="m-hd3"><h3 class="tit">热门标签</h3> </div>
                ...
            </div>
            <!-- 最热TOP5 -->
            <div class="sideblk">
                <div class="m-hd3"><h3 class="tit">最热TOP5</h3> <a href="#" class="s-fc02 f-fr">更多»</a></div>
                ...
            </div>
        </div>
    </div>
</div>
<!-- /侧栏内容区 -->
</body>
html结构优化项
  • 并列书写优于嵌套。

    如果可以写成<div></div><div></div>那么就不要写成<div><div></div></div>

  • 在满足视觉和语义的要求下，尽量不要有额外的冗余的结构。

    比如<div><h2></h2></div>已经能满足要求，那么就不要再写成<div><div><h2></h2></div></div>

  • 对于一个语义化的内部标签，应尽量避免使用className。

    比如在这样一个列表中，li标签中的itm应去除：<ul class="m-help"><li class="itm"></li><li class="itm"></li></ul>
