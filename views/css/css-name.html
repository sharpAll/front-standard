CSS规范 - 命名规则

使用类选择器，放弃ID选择器

ID在一个页面中的唯一性导致了如果以ID为选择器来写CSS，就无法重用。

在项目中，尽量使用类与样式关联，ID与数据和交互关联。

编码设置

如采用 UTF-8 编码，在 CSS 代码头部使用@charset "utf-8"

注意，必须要定义在 CSS 文件所有字符的前面（包括编码注释），@charset 才会生效。

不建议使用下划线 _ 进行连接

连接符统一使用'-'，节省操作，输入的时候少按一个 shift 键

能良好区分 JavaScript 变量命名

命名短且语义化良好

对于选择器的命名，尽量简洁且具有语义化，不应该出现 g-abc 这种语义模糊的命名。

相同语义的不同类命名

方法：直接加数字或字母区分即可（如：.m-list、.m-list2、.m-list3等，都是列表模块，但是是完全不一样的模块）。

其他举例：.g-fw0、.g-fw1、.s-cur0、.s-cur1、.m-logo2、.m-logo3、m-btn、m-btn2等等。

防止污染和被污染

当模块之间互相嵌套，且使用了相同的标签选择器或其他后代选择器，那么里面的选择器就会被外面相同的选择器所影响。

所以，如果你的模块可能嵌套或被嵌套于其他模块，那么要慎用标签选择器，必要时采用类选择器，并注意命名方式，可以采用.m-layer .layerxxx、.m-list2 .list2xxx的形式来降低后代选择器的污染性。
